# Database Schema

MongoDB (via Mongoose). Ten collections. All `_id` fields are MongoDB `ObjectId`. Timestamps
are stored as `createdAt` (`Date`, default now).

The original six model the marketplace (User, ServiceProvider, Service, ServiceRequest,
Project, Review); the last four add CRM features (Message, Notification, Task, Note).

## Entity relationships

```
User (1) ───────< Service (provider)
  │  │
  │  └─(role=provider, 1:1)─ ServiceProvider
  │
  ├─(customer)─< ServiceRequest >─(service)─ Service
  │                   │
  │                   └─(1:1)─ Project ──< updates[]
  │
  └─(customer)─< Review >─(provider / service / request)
```

- Registering with `role: "provider"` auto-creates a `ServiceProvider` profile (1:1 with `User`).
- Creating a `ServiceRequest` auto-creates a linked `Project` (1:1).
- A `Review` references the customer, provider, service, and the originating request.

## User
| Field | Type | Notes |
|-------|------|-------|
| `name` | String | required |
| `email` | String | required, unique, lowercased |
| `password` | String | required, min 6, **bcrypt-hashed** on save |
| `role` | String | enum `customer` \| `provider` \| `admin` (default `customer`) |
| `avatar` | String | Cloudinary URL (default `""`) |
| `createdAt` | Date | |

Instance method `matchPassword(plain)` compares against the hash.

|-------|------|-------|

| `bio` | String | |
| `skills` | [String] | |
| `experience` | [{ title, company, from, to, current, description }] | |
| `pricing` | { startingFrom: Number, currency: String=`USD` } | |
| `portfolio` | [{ title, description, image, link }] | `image` = Cloudinary URL |
| `rating` | Number | average rating (default 0) |

## Service
| Field | Type | Notes |
|-------|------|-------|
| `provider` | ObjectId → User | required |

| `description` | String | required |

## ServiceRequest
| Field | Type | Notes |
|-------|------|-------|
| `customer` | ObjectId → User | required |
| `service` | ObjectId → Service | required |
| `provider` | ObjectId → User | required |
| `requirements` | String | required |
| `budget` | Number | required |
| `deadline` | Date | required |

## Project
| Field | Type | Notes |
|-------|------|-------|
| `request` | ObjectId → ServiceRequest | required |
| `customer` | ObjectId → User | required |
| `provider` | ObjectId → User | required |
| `service` | ObjectId → Service | required |
| `status` | String | enum `pending` \| `accepted` \| `in-progress` \| `completed` \| `delivered` (default `pending`) |
| `updates` | [{ text: String, addedBy: ObjectId → User, createdAt: Date }] | progress timeline |
| `createdAt` | Date | |

**Transitions:** pending→accepted→in-progress→completed (provider) → delivered (customer).

## Review
| Field | Type | Notes |
|-------|------|-------|
| `customer` | ObjectId → User | required |
| `provider` | ObjectId → User | required |
| `service` | ObjectId → Service | required |
| `request` | ObjectId → ServiceRequest | required |
| `rating` | Number | required, 1–5 |
| `feedback` | String | default `""` |
| `createdAt` | Date | |

On create, the provider's `ServiceProvider.rating` and `numReviews` are recomputed across all
their reviews. One review per (request, customer) is enforced in the controller.

# CRM collections

## Message
A project-scoped 1:1 message between the project's customer and provider.

| Field | Type | Notes |
|-------|------|-------|
| `project` | ObjectId → Project | required |
| `sender` | ObjectId → User | required |
| `recipient` | ObjectId → User | required (the other party) |
| `text` | String | required, trimmed, max 2000 |
| `read` | Boolean | default `false` (set true when the recipient loads the thread) |
| `createdAt` | Date | |

Indexes: `{ project, createdAt }` (thread fetch), `{ recipient, read }` (unread lookups).

## Notification
An in-app alert addressed to one user, generated by platform events.

| Field | Type | Notes |
|-------|------|-------|
| `user` | ObjectId → User | required (recipient) |
| `type` | String | enum `request` \| `status` \| `message` \| `review` \| `task` \| `system` |
| `title` | String | required |
| `body` | String | default `""` |
| `link` | String | client route to open on click (e.g. `/projects/<id>`) |
| `read` | Boolean | default `false` |
| `createdAt` | Date | |

Generated by: new request (→ provider), request accepted/rejected (→ customer), project
status change (→ other party), new message (→ recipient), new review (→ provider).

## Task
A private follow-up reminder a party keeps on a project (owned by its creator).

| Field | Type | Notes |
|-------|------|-------|
| `project` | ObjectId → Project | required |
| `createdBy` | ObjectId → User | required (only the owner sees/edits it) |
| `title` | String | required, max 200 |
| `description` | String | default `""`, max 1000 |
| `dueDate` | Date | optional |
| `completed` | Boolean | default `false` |
| `createdAt` | Date | |

## Note
A private CRM note (with tags) one user keeps about another (a "contact").

| Field | Type | Notes |
|-------|------|-------|
| `author` | ObjectId → User | required (only the author can read/edit) |
| `subject` | ObjectId → User | required (the contact the note is about) |
| `body` | String | default `""`, max 2000 |
| `tags` | [String] | de-duped, trimmed, max 10 |
| `createdAt` / `updatedAt` | Date | |
